# API Reference

Complete API documentation for Nebula Logger's public classes and methods.

## Logger Class

The main entry point for logging in Apex.

### System Information Methods

#### `getVersionNumber()`
```apex
global static String getVersionNumber()
```
Returns the current version of Nebula Logger.

**Returns:** Version string in format `v0.0.0` (e.g., `v4.18.0`)

**Example:**
```apex
String version = Logger.getVersionNumber();
System.debug('Logger version: ' + version);
```

---

#### `getTransactionId()`
```apex
global static String getTransactionId()
```
Returns the unique ID for the current transaction.

**Returns:** UUID generated by Nebula Logger

**Example:**
```apex
String txnId = Logger.getTransactionId();
System.debug('Transaction ID: ' + txnId);
```

---

#### `getRequestId()`
```apex
public static String getRequestId()
```
Returns the request ID generated by Salesforce.

**Returns:** Value from `System.Request.getCurrent().getRequestId()`

---

#### `getCurrentQuiddity()`
```apex
global static System.Quiddity getCurrentQuiddity()
```
Returns the execution context of the current transaction.

**Returns:** `System.Quiddity` enum value

**Example:**
```apex
System.Quiddity context = Logger.getCurrentQuiddity();
if (context == System.Quiddity.BATCH_APEX) {
    // Batch-specific logging
}
```

### Settings Management

#### `isEnabled()`
```apex
global static Boolean isEnabled()
```
Checks if logging is enabled for the current user.

**Returns:** `true` if `LoggerSettings__c.IsEnabled__c == true`

**Example:**
```apex
if (Logger.isEnabled()) {
    Logger.info('Logging is enabled');
    Logger.saveLog();
}
```

---

#### `isEnabled(System.LoggingLevel)`
```apex
global static Boolean isEnabled(System.LoggingLevel loggingLevel)
```
Checks if a specific logging level is enabled.

**Parameters:**
- `loggingLevel` - The log level to check

**Returns:** `true` if the level meets or exceeds user's configured level

**Example:**
```apex
if (Logger.isEnabled(System.LoggingLevel.DEBUG)) {
    Logger.debug('Debug logging is enabled');
    Logger.saveLog();
}
```

---

#### `getUserLoggingLevel()`
```apex
global static System.LoggingLevel getUserLoggingLevel()
```
Gets the current user's configured logging level.

**Returns:** `System.LoggingLevel` from `LoggerSettings__c.LoggingLevel__c`

**Example:**
```apex
System.LoggingLevel level = Logger.getUserLoggingLevel();
System.debug('User log level: ' + level);
```

---

### Async Context Management

#### `setAsyncContext(Database.BatchableContext)`
```apex
global static void setAsyncContext(Database.BatchableContext batchableContext)
```
Stores batch job context information.

**Parameters:**
- `batchableContext` - The batch context from `start()` or `execute()` method

**Example:**
```apex
public class MyBatch implements Database.Batchable<SObject> {
    public Database.QueryLocator start(Database.BatchableContext bc) {
        Logger.setAsyncContext(bc);
        Logger.info('Batch started');
        Logger.saveLog();
        return Database.getQueryLocator('SELECT Id FROM Account');
    }
}
```

### Parent-Child Log Relationships

#### `setParentLogTransactionId(String)`
```apex
global static void setParentLogTransactionId(String parentTransactionId)
```
Links the current log to a parent log via `Log__c.ParentLog__c`.

**Parameters:**
- `parentTransactionId` - Transaction ID of the parent log

**Example:**
```apex
public class MyBatch implements Database.Batchable<SObject>, Database.Stateful {
    private String parentTxnId;
    
    public Database.QueryLocator start(Database.BatchableContext bc) {
        this.parentTxnId = Logger.getTransactionId();
        Logger.info('Batch started');
        Logger.saveLog();
        return Database.getQueryLocator('SELECT Id FROM Account');
    }
    
    public void execute(Database.BatchableContext bc, List<Account> scope) {
        Logger.setParentLogTransactionId(this.parentTxnId);
        Logger.info('Processing ' + scope.size() + ' accounts');
        Logger.saveLog();
    }
}
```

---

#### `getParentLogTransactionId()`
```apex
global static String getParentLogTransactionId()
```
Returns the parent transaction ID if set.

**Returns:** Parent transaction ID or `null`

### Custom Field Mapping

#### `setField(Schema.SObjectField, Object)`
```apex
global static void setField(Schema.SObjectField field, Object fieldValue)
```
Sets a custom field value on all `LogEntryEvent__e` records in the transaction.

**Parameters:**
- `field` - The field token (e.g., `LogEntryEvent__e.MyCustomField__c`)
- `fieldValue` - The value to set

**Example:**
```apex
Logger.setField(LogEntryEvent__e.MyCustomField__c, 'Custom Value');
Logger.info('This entry will have the custom field populated');
Logger.saveLog();
```

---

#### `setField(Map<Schema.SObjectField, Object>)`
```apex
global static void setField(Map<Schema.SObjectField, Object> fieldToValue)
```
Sets multiple custom field values.

**Parameters:**
- `fieldToValue` - Map of fields to values

**Example:**
```apex
Logger.setField(new Map<Schema.SObjectField, Object>{
    LogEntryEvent__e.CustomField1__c => 'Value 1',
    LogEntryEvent__e.CustomField2__c => 123
});
```

### Transaction Control

#### `suspendSaving()`
```apex
global static void suspendSaving()
```
Pauses saving for the current transaction. Calls to `saveLog()` are ignored until `resumeSaving()` is called.

**Example:**
```apex
Logger.suspendSaving();
for (Account acc : accounts) {
    Logger.debug('Processing ' + acc.Name); // Won't be saved
}
Logger.resumeSaving();
Logger.info('All accounts processed'); // Will be saved
Logger.saveLog();
```

---

#### `resumeSaving()`
```apex
global static void resumeSaving()
```
Resumes saving after `suspendSaving()`.

---

#### `isSavingSuspended()`
```apex
global static Boolean isSavingSuspended()
```
Checks if saving is currently suspended.

**Returns:** `true` if saving is suspended

---

#### `getBufferSize()`
```apex
global static Integer getBufferSize()
```
Returns the number of log entries in the buffer (not yet saved).

**Returns:** Count of buffered entries

**Example:**
```apex
Logger.info('Entry 1');
Logger.info('Entry 2');
Integer count = Logger.getBufferSize(); // Returns 2
```

---

#### `flushBuffer()`
```apex
global static void flushBuffer()
```
Discards all buffered log entries without saving them.

**Example:**
```apex
Logger.info('Entry 1');
Logger.info('Entry 2');
Logger.flushBuffer(); // Entries discarded
Logger.saveLog(); // Nothing to save
```

### Save Method Management

#### `setSaveMethod(SaveMethod)`
```apex
global static void setSaveMethod(SaveMethod saveMethod)
```
Sets the default save method for subsequent `saveLog()` calls.

**Parameters:**
- `saveMethod` - The `Logger.SaveMethod` enum value

**Example:**
```apex
Logger.setSaveMethod(Logger.SaveMethod.QUEUEABLE);
Logger.info('Entry 1');
Logger.saveLog(); // Uses QUEUEABLE

Logger.info('Entry 2');
Logger.saveLog(); // Still uses QUEUEABLE
```

### Save Methods

#### `saveLog()`
```apex
global static void saveLog()
```
Saves all buffered log entries using the default save method.

**Example:**
```apex
Logger.info('Entry 1');
Logger.info('Entry 2');
Logger.saveLog();
```

---

#### `saveLog(SaveMethod)`
```apex
global static void saveLog(SaveMethod saveMethod)
```
Saves all buffered log entries using the specified save method.

**Parameters:**
- `saveMethod` - The save method to use for this call only

**Example:**
```apex
Logger.info('Entry 1');
Logger.saveLog(Logger.SaveMethod.REST);
```

---

#### `saveLog(String)`
```apex
global static void saveLog(String saveMethodName)
```
Saves using a save method specified by name.

**Parameters:**
- `saveMethodName` - Name of the save method ('EVENT_BUS', 'QUEUEABLE', 'REST', 'SYNCHRONOUS_DML')

---

## SaveMethod Enum

```apex
global enum SaveMethod {
    EVENT_BUS,           // Default: async via platform events
    QUEUEABLE,           // Async via queueable job
    REST,                // Sync via REST API (avoids mixed DML)
    SYNCHRONOUS_DML      // Sync DML (careful: rolls back on error)
}
```

**Usage:**
```apex
Logger.saveLog(Logger.SaveMethod.QUEUEABLE);
```

## LogEntryEventBuilder Class

Fluent API for building log entries. Returned by all logging methods.

### Methods

#### `setMessage(String)`
```apex
global LogEntryEventBuilder setMessage(String message)
```
Updates the log entry message.

---

#### `setRecord(Id)`
```apex
global LogEntryEventBuilder setRecord(Id recordId)
```
Associates a record with the log entry via ID only.

---

#### `setRecord(SObject)`
```apex
global LogEntryEventBuilder setRecord(SObject record)
```
Associates a record with the log entry, capturing field values.

**Example:**
```apex
Account acc = [SELECT Id, Name, AnnualRevenue FROM Account LIMIT 1];
Logger.info('Processing account')
    .setRecord(acc); // Captures all fields from the query
```

---

#### `setRecord(List<SObject>)`
```apex
global LogEntryEventBuilder setRecord(List<SObject> records)
```
Associates multiple records.

---

#### `setExceptionDetails(Exception)`
```apex
global LogEntryEventBuilder setExceptionDetails(Exception ex)
```
Captures exception details (type, message, stack trace).

**Example:**
```apex
try {
    // risky code
} catch (Exception e) {
    Logger.error('Operation failed')
        .setExceptionDetails(e)
        .addTag('exception');
    Logger.saveLog();
}
```

---

#### `addTag(String)`
```apex
global LogEntryEventBuilder addTag(String tag)
```
Adds a tag to the log entry.

**Example:**
```apex
Logger.info('Payment processed')
    .addTag('payment')
    .addTag('critical')
    .addTag('audit-required');
```

---

#### `addTags(List<String>)`
```apex
global LogEntryEventBuilder addTags(List<String> tags)
```
Adds multiple tags.

**Example:**
```apex
Logger.info('Event occurred')
    .addTags(new List<String>{'tag1', 'tag2', 'tag3'});
```

---

#### `setField(Schema.SObjectField, Object)`
```apex
global LogEntryEventBuilder setField(Schema.SObjectField field, Object value)
```
Sets a custom field value on this specific log entry.

---

#### `setField(Map<Schema.SObjectField, Object>)`
```apex
global LogEntryEventBuilder setField(Map<Schema.SObjectField, Object> fieldToValue)
```
Sets multiple custom field values.

---

#### `shouldSave()`
```apex
global LogEntryEventBuilder shouldSave()
```
Forces the entry to be saved regardless of log level.

**Example:**
```apex
Logger.finest('Very detailed info')
    .shouldSave(); // Saves even if user level is INFO
```

---

#### `shouldNotSave()`
```apex
global LogEntryEventBuilder shouldNotSave()
```
Prevents the entry from being saved.

### Batch Job with Parent-Child Logs
```apex
public class MyBatch implements Database.Batchable<SObject>, Database.Stateful {
    private String parentTxnId;
    
    public Database.QueryLocator start(Database.BatchableContext bc) {
        Logger.setAsyncContext(bc);
        this.parentTxnId = Logger.getTransactionId();
        Logger.info('Batch started');
        Logger.saveLog();
        return Database.getQueryLocator('SELECT Id FROM Account');
    }
    
    public void execute(Database.BatchableContext bc, List<Account> scope) {
        Logger.setAsyncContext(bc);
        Logger.setParentLogTransactionId(this.parentTxnId);
        Logger.info('Processing ' + scope.size() + ' records');
        // ... processing logic ...
        Logger.saveLog();
    }
    
    public void finish(Database.BatchableContext bc) {
        Logger.setAsyncContext(bc);
        Logger.setParentLogTransactionId(this.parentTxnId);
        Logger.info('Batch completed');
        Logger.saveLog();
    }
}
```
